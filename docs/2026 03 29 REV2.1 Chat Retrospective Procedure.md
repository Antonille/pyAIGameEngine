# 2026 03 29 REV2.1 Chat Retrospective Procedure

## Purpose
`chat retrospective` is the named procedure for extracting reusable workflow/process lessons from a completed chat block and pushing those lessons into the current project operating docs.

## When to use it
Use `chat retrospective` when a just-completed block produced meaningful lessons about:
- prompt structure
- workflow efficiency
- validation discipline
- benchmark interpretation
- relay timing
- interface/control-boundary timing
- handoff clarity

## Expected outputs
1. updated lessons learned
2. updated workflow/resource docs
3. updated optimized prompt templates
4. any needed current-pointer updates
5. identified POWE promotion candidates
6. concise summary of new lessons
7. recommended next prompt

## Soft-trigger policy
Like project relay, this should be suggested after a meaningful block, not forced as an interruption.
