## Durable prompt structure
The most effective prompts in this project usually separate:
1. current-truth anchor
2. pass type
3. constraints / deferrals
4. required outputs
5. exact validation expectations
6. next-step handoff

## Analysis/report-regeneration pass template
Use the current pyAIGameEngine project state as truth.

Pass type:
Analysis/report-regeneration pass.

Task:
Validate uploaded archival test outputs and generated artifacts, integrate accepted testing truth into the current project state, regenerate the current report from archival data, and refresh the relevant docs and prompts.

Requirements:
1. Review the current-pointer docs first.
2. Treat archival test data as append-only durable truth.
3. Treat the current human-readable report as regenerable.
4. Preserve fallback-first architecture and PyBullet deferral unless validated results change that truth.
5. Distinguish what is newly accepted, what remains provisional, and what should not yet be overclaimed.
6. Update workflow, lessons, and next-step prompts when accepted testing truth changes project guidance.
7. Deliver updated report artifacts, updated docs, concise conclusions, exact local copy-in instructions, and the next recommended prompt.
