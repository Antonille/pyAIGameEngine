# 2026 03 30 REV4.0 Lessons Learned

## Purpose
This document consolidates durable architecture, tooling, benchmarking, workflow, prompting, packaging, and relay lessons that should remain easy to recover across chats.

## Architecture and tooling lessons already in force
- Separate proof-of-concept folders reduce accidental cross-talk and make architectural comparison easier.
- The canonical engine-state decision matters early; SoA arrays are the right default experiment for throughput-oriented physics and AI.
- Baseline the project on **Python 3.12.x** for now to reduce package friction.
- A single `.venv` is simpler than split environments at this stage.
- The engine architecture should preserve a fallback backend path so core SoA / plank-tic work can continue even when optional native dependencies are unavailable.

## Runtime and benchmark lessons already in force
- A standalone kernel speedup does not guarantee an integrated runtime speedup.
- Fair NumPy vs Numba comparison requires warmup awareness and structured benchmark output.
- Integrated benchmark/runtime truth should outweigh isolated microbenchmark wins when representative evidence is available.
- Smoke runs are for instrumentation visibility; representative runs should drive benchmark claims.

## Relay and truth-establishment additions from this chat block
### Candidate-first truth lesson
- When a candidate full snapshot is available, use that candidate as the truth anchor before consulting older sparse artifacts.
- Sparse passes may contain newer docs, but they must be reconciled against candidate code truth rather than treated as independent project truth.

### ZIP packaging lesson
- Every full and sparse snapshot should include a root-level `OPEN_THIS_FIRST.md` and `SNAPSHOT_INDEX.json` so project truth can be established quickly and deterministically.
- The package should also include `docs/CURRENT_ACTIVE_FILE_PATHS.md` so the current working set does not depend on conversational memory.

### Python-first ZIP handling lesson
- ChatGPT should use **Python first** when inspecting ZIP files for truth establishment. Do not rely on non-Python ZIP opening paths first.

### Snapshot-script compatibility lesson
- Snapshot scripts must be validated under the actual target PowerShell/runtime.
- Avoid:
  - `[System.IO.Path]::GetRelativePath` on Windows PowerShell 5.1
  - parameter-default logic that depends on `$PSScriptRoot`
  - `TrimStart` slash-cleanup patterns that rely on fragile char conversion

### Docs-vs-code truth lesson
- Documentation can get ahead of merged code truth. Relay reconciliation must explicitly check that current docs do not imply code state that is absent from the candidate snapshot.

## Prompt and workflow takeaways
- Prompts are more effective when the pass type is explicit.
- The most reliable prompts combine:
  - current-truth anchor
  - constraints/deferrals
  - exact deliverables
  - exact validation commands
  - next-step handoff
- Relay suggestions are better as soft triggers than forced interruptions.
