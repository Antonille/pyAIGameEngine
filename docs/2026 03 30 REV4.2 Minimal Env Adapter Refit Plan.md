# 2026 03 30 REV4.2 Minimal Env Adapter Refit Plan

## Purpose
This document defines the intended target shape for refitting the current minimal environment adapter toward the packet boundary without prematurely freezing a public RL API.

## Decided now
- The current minimal Gym-compatible path should be treated as temporary scaffolding.
- The refit target is a thin project-local adapter that:
  - stages runtime-schema-driven observation blocks from SoA-authoritative state
  - hands action intent to the packet path
  - relies on action-bridge application to perform authoritative writes
  - keeps train/eval/benchmark mode explicit
  - records seed and mode as first-class metadata
- The adapter should stay thin and should not become the place where engine-side model logic accumulates.

## Temporary scaffolding
- Current direct writes into `state.action` are temporary shortcut behavior.
- The adapter may still reuse some existing POC-local observation or rollout helpers while the packet-bound refit is planned.
- A narrow Gym-compatible wrapper is acceptable as an outer shell during transition, provided the internal contract direction shifts toward packet flow.

## Deferred
- public RL API hardening
- many-world execution support
- broad wrapper ecosystem support
- framework-specific environment abstractions inside the engine core

## Open questions
- What is the minimum viable refit that proves the packet-bound adapter direction without creating large churn?
- Should the first refit target benchmark mode, eval mode, or train mode first?
- Which helper logic should live in the adapter versus shared controller utilities?

## Intended packet-oriented flow
### Decided now
Target shape:
1. authoritative SoA state advances on the plank-tic loop
2. contract-facing feature blocks are staged from SoA-authoritative state
3. adapter exposes contract-facing observations outward
4. controller / policy / model returns action intent
5. adapter packages action intent into action packets
6. action-bridge applies packet semantics
7. authoritative `state.action` is written
8. application/physics stage consumes authoritative action state

### Temporary scaffolding
- Current direct `state.action` mutation is transitional and should be reduced over time.

### Deferred
- heterogeneous packet translation layers for many model families at once
- many-world packet fan-out logic

### Open questions
- Which existing packet builders and action-bridge utilities can be reused directly by the first refit?
- What is the smallest benchmarkable adapter scenario that proves the refit direction?

## Observation staging location
### Decided now
- Observation staging should occur outside the hot simulation core, but close enough to authoritative state access that it does not grow heavy Python object churn.
- Runtime-schema-driven feature-block staging is the desired long-term observation path.

### Temporary scaffolding
- POC-local observation builders may continue temporarily if they are explicitly treated as transitional and aligned with the feature-block contract direction.

### Deferred
- full zero-copy guarantees
- richer GPU-oriented staging guarantees

### Open questions
- What is the first observation staging boundary that is explicit enough to stabilize future adapter work?
