# 2026 03 31 REV4.6 Lessons Learned

## Purpose
Capture durable project and workflow lessons after the cleanup promotion and the follow-on contamination-control clarification pass.

## Best practices strengthened
- Keep the active repo/documentation surface intentionally small.
- Archive superseded revision series instead of letting them compete with current truth.
- Remove transient/editor/runtime byproducts before publishing snapshots or repo truth.
- Preserve the lesson-action register as the thin implementation layer on top of narrative lessons.
- Treat full snapshots as authoritative only after reconciliation against later sparse work.

## New lessons from the post-promotion pass
### Best practice now adopted
Canonical environment-variable names must be treated as part of project truth. Scripts and docs should use a single approved name for each path and fail fast when a required variable is missing.

### Best practice now adopted
Line-ending policy should be repo-owned, not assumed. A committed `.gitattributes` file plus a one-time renormalization pass is preferable to relying on cross-platform defaults.

### Best practice now adopted
Chat copy/paste is not a trustworthy transport for durable repository files. Downloaded artifacts, ZIP packages, or direct file writes are safer and more restart-friendly.

### Best practice now adopted
CRLF/LF warnings should not be ignored as background noise. They should either be expected under an explicit policy or treated as a signal that file transport or repo normalization needs attention.

## Open risks still visible
- latest-snapshot selector hardening remains unfinished
- line-ending policy is documented but still needs a committed `.gitattributes` implementation pass unless already added later
- some Windows script documentation still needs to be refreshed alongside the actual script rewrite outcome
- if pasted text is committed without editor review, avoidable churn can still enter repo-truth
