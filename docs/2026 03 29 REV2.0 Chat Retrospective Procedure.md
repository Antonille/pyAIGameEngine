# 2026 03 29 REV2.0 Chat Retrospective Procedure

## Purpose
`chat retrospective` is the named procedure for extracting and preserving reusable workflow/process lessons from a completed or nearly completed chat block.

## When to use it
Use `chat retrospective` when:
- the chat produced reusable workflow or prompting lessons
- validation or benchmark interpretation changed how future passes should be run
- interface/control-boundary timing lessons became clearer
- the project has accumulated enough new process knowledge that future chats would benefit from updated docs and templates

## Core outputs
A successful `chat retrospective` should produce:
1. refreshed lessons learned
2. refreshed workflow/resource docs
3. refreshed optimized prompts/templates
4. any needed current-pointer updates
5. a short list of lessons to promote into the Prompt Optimization and Workflow Engineering knowledge base
6. a concise recommended next prompt

## Relationship to `project relay`
`chat retrospective` does not replace `project relay`.

Typical relationship:
- use `chat retrospective` to preserve workflow/process lessons
- use `project relay` to package the broader project state for transfer to a new chat

Either may happen first depending on what was learned and whether a handoff is imminent.

## Recommended prompt skeleton
Use the current pyAIGameEngine project state as truth.

Task:
Perform the named procedure **chat retrospective**.

Requirements:
1. Extract additional workflow, prompting, validation, relay, benchmark-interpretation, and interface-timing lessons from the completed chat block.
2. Update the relevant docs and prompt artifacts.
3. Identify which lessons should also be promoted into the Prompt Optimization and Workflow Engineering knowledge base.
4. Deliver updated docs, a concise summary, and the next recommended prompt.
