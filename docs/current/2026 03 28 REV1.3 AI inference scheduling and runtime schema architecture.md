# 2026 03 28 REV1.3 AI inference scheduling and runtime schema architecture

## Accepted direction
This document consolidates the current accepted guidance for AI scheduling and runtime-schema design.

## Runtime schema direction
- Use fixed per-family runtime schemas.
- Keep CPU-authoritative SoA storage for active POC work.
- Define feature block schemas per model family.
- Use precomputed feature-pack plans rather than ad hoc field fetching.
- Track freshness/version by block, not per scalar.

## Scheduling direction
- Schedule by model family, not by individual agent call site.
- Each model family defines cadence, max input staleness, max output staleness, stage deadline, and fallback policy.
- Plan feature movement by packed block.
- Use CPU pinned feature-pack buffers for staged transfer if/when GPU inference is introduced.
- Overlap CPU work with inference whenever possible.

## Current implementation posture
- The architecture direction is adopted.
- Full implementation is still partial.
- The next practical step is a small reusable CPU-side scheduling seam for one repeated object family.
