## Named procedures
### validate and update
Use when you upload a new local snapshot and want it reconciled, versioned, documented, and returned as a sparse or full updated snapshot.

### full consolidate
Use when you want the project fully refreshed: docs aligned, archives updated, clean snapshot produced, and tracking files rolled forward.

### sparse refresh
Use when only changed files should be returned, along with modification-history updates and copy-in instructions.

### sanity reconcile
Use when you want a consistency check between the uploaded local state and expected project truth before changes are made.

### benchmark compare
Use when you want a structured comparison, typically NumPy vs Numba, with timing summaries prepared for docs and manifests.

### promote candidate
Use when a POC-local component may be ready to move into shared engine space.

### doc currentize
Use when you only want docs/tracking/prompt artifacts aligned to current code reality.
