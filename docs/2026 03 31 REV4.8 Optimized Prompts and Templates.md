# 2026 03 31 REV4.8 Optimized Prompts and Templates

## Durable prompt structure
The most effective prompts in this project continue to separate:
1. truth anchor
2. pass type
3. constraints / deferrals
4. required outputs
5. exact validation expectations
6. next-step handoff

## Additional contamination-control requirement
For repo-maintenance, consolidation, script, docs, testing-analysis, or report-generation passes, explicitly state:
- whether the pass is using repo-truth or snapshot-truth
- which canonical local environment variables should be used
- whether the work should be delivered as downloadable file artifacts or pasted text
- whether line-ending policy is relevant to the touched files

## Durable transport rule for prompts
If the pass creates or edits durable repo files:
- prefer downloadable file artifacts or ZIP bundles
- do not rely on long raw chat copy/paste when file bytes or line endings matter
- if paste is unavoidable, instruct the user to review EOL state in an editor before commit

## Testing-analysis/report prompt rule
If the pass is primarily about testing analysis, current report generation, or report-surface cleanup:
- explicitly name `docs/CURRENT_TEST_REPORT_PROCEDURE.md`
- do not rely only on current file presence
- require distinction between archival durable truth and current regenerable reports
- require artifact classification where legacy or auxiliary visuals may coexist with authoritative current outputs

Use GitHub main as repo-truth for pyAIGameEngine and use the synced local repository plus the actual Windows `.venv` as the execution target when required.

Pass type:
Testing analysis and report pass.

Requirements:
1. Read `docs/CURRENT_TEST_REPORT_PROCEDURE.md`.
