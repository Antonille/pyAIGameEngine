# docs overview

## How to use this folder
1. Start with `../OPEN_THIS_REPO_FIRST.md`.
2. Start with `CURRENT_*` pointer files.
3. Read the active revision docs those pointers resolve to.
4. Use `ARCHIVE_INDEX.md` when you need older history or superseded materials.
5. Use `docs/research/` for architecture research inputs that are informative but not automatically active truth until promoted.

## Active-surface policy
The docs root is intentionally curated.
It should contain the current pointers, the active revision targets, and only a few organizing docs.
Historical revision series belong under `docs/archive/`.
