# 2026 03 30 REV3.8 Artifact and Versioning Plan

## Purpose
This document defines the lightweight artifact/versioning discipline needed before pyAIGameEngine widens into broader experiment, simulation, and AI/RL work.

## Artifact classes currently in scope
- candidate snapshots
- benchmark configurations
- scenario definitions / bundles
- archival test history
- current/generated test reports
- model artifacts later
- hyperparameter / experiment configs later
- adapter-facing run metadata later

## Research-aligned additions now recommended
The artifact system should support future coexistence with ASV/DVC/MLflow-style metadata without requiring those tools now.

Recommended archival metadata additions:
- machine/environment fingerprint
- benchmark compatibility group key
- run classification (`smoke`, `representative`, `acceptance`, `regression`)
- stable scenario/seed identifiers
- artifact references for generated plots/images/report outputs

## Current recommendation
Do not build a heavy artifact-management system yet.
Do build a durable append-only archival truth model now so that later ASV/DVC/MLflow adoption is optional, not required for recovering project history.
