## Durable prompt structure
The most effective prompts in this project continue to separate:
1. truth anchor
2. pass type
3. constraints / deferrals
4. required outputs
5. exact validation expectations
6. next-step handoff

## Additional configurable-interface requirement
For interface-boundary work, explicitly state:
- what remains simulation-authoritative
- what is interface-facing
- what is canonical
- what is generated/derived
- what is intentionally deferred
- what must be validated and compiled at startup
- what must remain fixed/indexed in the hot path

## Implementation pass template addendum
When implementing configurable-interface work:
1. preserve fallback-first architecture and PyBullet deferral
2. do not replace the hot-path mapping until the startup-compiled subset is proven
3. keep the readable interface doc and validator synchronized
4. leave exact local validation commands for manifest validation and runtime checks
5. update the testing/reporting docs if interface metadata becomes archival/report content
