# 2026 03 31 REV4.9 Current Workflow

## Current phase
GitHub-first workflow active on top of the fallback-first POC baseline, with accepted archival testing truth, startup-compiled configurable-interface groundwork, and a curated documentation surface promoted into repo-truth.

## Default work cadence
1. Choose the **pass type**.
2. Anchor the work to the best available **current truth**.
3. State what should **not** be overcommitted yet.
4. Perform **pre-pass retrieval**:
   - read relevant recent lessons from `docs/CURRENT_LESSONS_LEARNED.md`
   - read open items from `docs/CURRENT_LESSON_ACTION_REGISTER.md`
   - read `docs/CURRENT_CONFIGURABLE_INTERFACE_CONTROL.md` when interface-boundary work is involved
   - state which truth tier is being used for the pass
5. Execute the code/doc/spec pass.
6. Provide exact validation commands where relevant.
7. Flash-capture lesson rows during the pass after failed validation, workflow breakdown, interface clarification, or unexpected benchmark/runtime results.
8. End with retrospective synthesis and concrete-destination assignment.
9. Commit/push intended GitHub state when local truth is ready.
10. Export sparse/full snapshots only when they materially help recovery, relay, or rollback.

## Configurable-interface workflow rule
- The configurable interface must be defined in two synchronized places:
  1. a stakeholder-readable interface control document
  2. a programmatic validator/compile path
- Prefer one canonical manifest plus generated/refreshed readable docs over manual dual maintenance.
- The startup compile/freeze step is the performance boundary: flexibility is allowed at startup, not in the plank-tic loop.
- Simulation-authoritative structures remain separate from interface-facing structures.

## Current caution points
- Do not switch the hot-path runtime wholesale to dynamic interface handling yet.
- Do not freeze a public RL API around the current Gym path.
- Do not treat narrative lessons as implemented without evidence-backed follow-through.
- Do not let configurable-interface flexibility leak into per-step runtime cost.
- Do not let supporting local evidence silently replace repo-truth.
