## Truth tiers to use going forward
1. **Repo-truth** = public/private GitHub main state that is committed and pushed
2. **Snapshot-truth** = archival or transfer snapshot that is intentionally ahead of the repo for a bounded period
3. **Supporting local evidence** = uploaded logs, candidate files, temporary outputs, and local observations that may inform diagnosis but do not override repo-truth unless explicitly promoted

## Workflow heuristics
- Use a clearly named **pass type** whenever practical.
- Use **GitHub + current-pointer docs** as the shared truth anchor unless a newer local/uploaded artifact is explicitly ahead.
- Preserve fallback-first execution and PyBullet deferral unless validated work changes that truth.
- Keep simulation-authoritative structures separate from interface-facing structures.
- Treat archival JSONL test history as durable truth and current reports as regenerable derivatives.
- Do not freeze a public RL API around the current minimal Gym path.
- Keep workflow hardening lightweight and markdown-first.
- Turn lessons into one concrete destination:
  - script/tool fix
  - workflow checklist change
  - current-pointer doc update
  - prompt-template update
  - validation/test addition
  - deferred/not-yet list
  - backlog item
