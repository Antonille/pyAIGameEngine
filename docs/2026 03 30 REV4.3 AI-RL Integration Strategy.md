# 2026 03 30 REV4.3 AI-RL Integration Strategy

## Purpose
Define how AI and reinforcement learning should be integrated into pyAIGameEngine without destabilizing the engine and simulation foundations.

## Core strategy
AI/RL remains downstream of engine credibility and packet-oriented control boundaries.

## Decided now
- The project already has the beginnings of the right AI boundary in code.
- The main near-term risk is boundary drift, not lack of primitives.
- The next high-value implementation sequence is deterministic reset/replay and packet-oriented minimal adapter hardening.
- Runtime-schema-driven feature blocks staged from SoA-authoritative state remain the future-correct observation path.
- The authoritative action path remains packet-first.

## Temporary scaffolding
- The minimal Gym path is still temporary scaffolding.
- Mock inference payloads and narrow adapter helpers remain POC-local test tools.

## Deferred
- Public RL API hardening.
- Many-world rollout as an active milestone requirement.
- Model-framework lock-in.

## Open questions
- First stable adapter contract width and observation-block set.
- First stable replay-file or replay-bundle format beyond in-memory snapshot dictionaries.
