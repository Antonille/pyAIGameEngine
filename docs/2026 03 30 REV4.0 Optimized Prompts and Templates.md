# 2026 03 30 REV4.0 Optimized Prompts and Templates

## Durable prompt structure
The most effective prompts in this project usually separate:
1. current-truth anchor
2. pass type
3. constraints / deferrals
4. required outputs
5. exact validation expectations
6. next-step handoff

## ZIP-handling rule for snapshot prompts
When a ZIP snapshot is uploaded, use **Python first** to inspect and enumerate the ZIP contents.
Establish truth in this order:
1. `OPEN_THIS_FIRST.md`
2. `SNAPSHOT_INDEX.json`
3. current-pointer docs
4. manifest / modification history
5. active code files

Do not rely on non-Python ZIP opening paths first when establishing project truth.

## Documentation/interface pass template
Use the current pyAIGameEngine project state as truth.

Pass type:
Documentation/interface pass.

Task:
Refine the relevant control docs, developer guides, prompt artifacts, workflow docs, and current-truth pointers.

Requirements:
1. Keep the docs aligned with current validated truth.
2. Prefer clear interface/control boundaries before larger refactors.
3. Preserve restartability and handoff clarity.
4. Update current-pointer docs if needed.
5. If attached research or validation output exists, integrate it explicitly rather than re-deriving conclusions from memory.
6. If ZIP/snapshot workflow is touched, update `OPEN_THIS_FIRST.md`, `SNAPSHOT_INDEX.json`, and `docs/CURRENT_ACTIVE_FILE_PATHS.md` design/requirements as needed.
7. Deliver updated docs and the next recommended prompt.

## Project relay template
Use the uploaded pyAIGameEngine candidate snapshot as the current project truth.

Before doing new implementation work:
1. Review `OPEN_THIS_FIRST.md` first.
2. Review `SNAPSHOT_INDEX.json` next.
3. Review the current-pointer docs.
4. Treat those files as the authoritative summary of project status, workflow, lessons learned, prompt artifacts, and current best practices.

Pass type:
Relay/handoff pass.

Task:
Perform the named procedure **project relay**.

Requirements:
1. Reconcile current chat truth against the latest candidate snapshot.
2. Refresh current-truth docs and current-pointer docs.
3. Refresh `OPEN_THIS_FIRST.md`, `SNAPSHOT_INDEX.json`, and `docs/CURRENT_ACTIVE_FILE_PATHS.md`.
4. Package the project cleanly for transfer to a new chat.
5. Deliver the snapshot, bootstrap prompt, local instructions, and next recommended prompt.
