# 2026 03 28 REV1.4 GitHub and Snapshot Workflow

## Intent
Keep the local repo, GitHub repo, and snapshot artifacts aligned with current project truth.

## Recommended local sequence
```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\windows\run_poc1_validation.ps1 -BackendMode numpy
powershell -ExecutionPolicy Bypass -File .\scripts\windows\run_poc1_validation.ps1 -BackendMode numba
powershell -ExecutionPolicy Bypass -File .\scripts\windows\update_repo.ps1 -Message "Update pyAIGameEngine to current validated state"
powershell -ExecutionPolicy Bypass -File .\scripts\windows\export_snapshot.ps1 -Revision 0.0.9
```

## Snapshot cleanliness
The export scripts should exclude transient files:
- `.venv`
- `__pycache__`
- `.pyc` / `.pyo`
- `.egg-info`
- `.pytest_cache`, `.mypy_cache`, `.ruff_cache`
- Numba cache artifacts (`.nbc`, `.nbi`)
- `build/`, `dist/`, `snapshots_out/`

## GitHub helper scripts
- `init_git_repo.ps1` should initialize the repo on the current revision and optionally create/push a private GitHub repo.
- `update_repo.ps1` should add, commit, and push with a meaningful message.
- `export_snapshot.ps1` should create a clean snapshot zip suitable for archival or upload.
