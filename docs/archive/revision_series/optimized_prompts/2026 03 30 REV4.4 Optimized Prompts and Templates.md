# 2026 03 30 REV4.4 Optimized Prompts and Templates

## Durable prompt structure
The most effective prompts in this project should explicitly separate:
1. **truth anchor**
2. **pass type**
3. **constraints / deferrals**
4. **required outputs**
5. **exact validation expectations**
6. **next-step handoff**

## Truth-tier language to use explicitly
When relevant, prompts should identify which of the following is authoritative for the pass:
- **repo-truth** = committed/pushed GitHub state
- **snapshot-truth** = intentionally ahead archival/transfer snapshot
- **supporting local evidence** = logs, uploaded files, candidate artifacts, temporary outputs, and diagnostic observations that do not override the chosen truth anchor unless explicitly promoted

## Pre-pass retrieval rule
Before implementation, testing, relay/handoff, and retrospective passes:
1. read the most relevant recent lessons from `docs/CURRENT_LESSONS_LEARNED.md`
2. read the open items in `docs/CURRENT_LESSON_ACTION_REGISTER.md`
3. note which items are being applied, deferred, or left as open risks

## Implementation pass template
Use the current pyAIGameEngine repo-truth as truth unless a newer snapshot-truth is explicitly supplied and adopted for the pass.

Pass type:
Implementation pass.

Task:
Implement the requested bounded change.

Requirements:
1. Preserve fallback-first architecture and keep PyBullet optional/deferred unless validated work changes that truth.
2. State which truth tier is authoritative for the pass.
3. Perform pre-pass retrieval of the relevant recent lessons and open action items.
4. Keep current benchmark/runtime truth intact unless explicitly superseded by new validated results.
5. Update code, docs, workflow artifacts, and prompts as needed.
6. Provide exact local validation commands and expected high-level outcomes.
7. Promote lessons to best practices only after validation evidence exists.
8. Provide the next recommended prompt.

## Documentation/interface pass template
Use the current pyAIGameEngine repo-truth as truth unless a newer snapshot-truth is explicitly supplied and adopted for the pass.

Pass type:
Documentation/interface pass.

Task:
Refine the relevant control docs, developer guides, prompt artifacts, and workflow docs.

Requirements:
1. Keep the docs aligned with current validated truth.
2. Perform pre-pass retrieval of the most relevant lessons and open action items.
3. Prefer clear interface/control boundaries before larger refactors.
4. Preserve restartability and handoff clarity.
5. Update current-pointer docs if needed.
6. Provide exact local validation commands where relevant.
7. Provide the next recommended prompt.

## Retrospective/process-improvement pass template
Use the current pyAIGameEngine repo-truth as truth unless a newer snapshot-truth is explicitly supplied and adopted for the pass.

Pass type:
Retrospective/process-improvement pass.

Task:
Perform the named procedure **chat retrospective**.

Requirements:
1. Perform pre-pass retrieval of the most relevant lessons and open action items.
2. Extract additional workflow, prompting, validation, relay, benchmark-interpretation, and interface-timing lessons from the completed chat block.
3. Update:
   - Project Resource Guide
   - Lessons Learned
   - Lesson Action Register
   - optimized prompts/templates
   - workflow docs
4. Distinguish clearly between:
   - newly observed lessons
   - adopted best practices
   - defer rules
   - open risks
5. Add or refine a short workflow-heuristics section where helpful.
6. Identify which lessons should also be promoted into the Prompt Optimization and Workflow Engineering knowledge base.
7. Deliver:
   - updated docs
   - concise summary of new lessons
   - exact validation commands where relevant
   - recommended next prompt

## Relay/handoff pass reminder
Every relay/handoff pass should state:
- which truth tier was authoritative
- whether GitHub is caught up or local truth is still ahead
- whether the current-pointer freshness audit was completed
- whether full+sparse reconciliation was completed if a new full snapshot is being declared authoritative
