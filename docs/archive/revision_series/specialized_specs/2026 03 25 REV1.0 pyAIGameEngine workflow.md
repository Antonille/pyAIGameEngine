# pyAIGameEngine Workflow — REV1.0

## Purpose
This workflow keeps the local project, repo history, snapshots, and ChatGPT project state aligned while the engine architecture is still fluid.

## Current phase
1. Conceptual design and vocabulary stabilization
2. Research into existing tools and packages
3. Early engine architecture definition
4. **Immediate target:** minimally functional POC1

## Working model
- Use isolated proof-of-concept folders for architecture experiments.
- Promote code out of a POC only after it proves reusable across future directions.
- Keep all important project state in revisioned markdown files under `docs/`.
- Prefer timestamped snapshots when pausing after meaningful progress.
- Every snapshot should carry workflow, progress, lessons learned, modification history, and next prompts.
- **Current environment baseline:** one local `.venv` on **Python 3.12.x**.

## Daily/iteration loop
1. Pull latest local truth into chat via snapshot upload.
2. Run consistency/sanity check against prior snapshot assumptions.
3. Apply scoped work for the current milestone.
4. Update revisioned docs:
   - workflow
   - progress notes
   - lessons learned
   - optimized prompts/templates
   - modification history
   - next-step prompts
   - POC workflow doc(s)
5. Run install/validation/packaging scripts as needed.
6. Commit locally and update GitHub.
7. Export a new snapshot ZIP when a pause point is reached.

## Snapshot rules
- Full snapshot: use at milestone boundaries or when many files changed.
- Sparse snapshot: use when changes are mostly docs/scripts or a contained feature area.
- Always bump revision numbers when a new snapshot becomes the working truth.
- Always update the tracking docs in the snapshot before delivery.

## Git/GitHub workflow
1. Initialize repo locally if needed.
2. Create or connect private GitHub repo.
3. Commit meaningful checkpoints with concise messages.
4. Tag milestone-adjacent commits.
5. Push after local validation.

## Best-practice checks before any snapshot
- POC folder runs without importing undeclared shared utilities.
- Shared scripts do not assume a single POC architecture.
- All revisioned docs have matching revision/date references.
- Next-step prompts reflect the new truth.
- Snapshot name matches project revision.
- Install docs and install scripts agree on **Python 3.12**.
