## Current winner claim with workload context
For the **ownership-aware `no_change` / baseline-ownership-targeted packet workload subset currently validated**:
- `20 / 64` is a sanity/instrumentation run
- `500 / 1024` is the current medium representative run

Current claim:
- **Warmed Numba is currently the integrated winner for the validated medium and long ownership-aware no-change-enabled packet workload subset.**
- The current advantage appears to come mainly from stage timings, especially physics, with a smaller favorable contribution from AI stage timing.
- Packet-layer metrics now show a cleaner workload shape because ownership-targeted semantics reduced explicit invalidate traffic.

## Key implications
- Ownership-aware packet semantics appear to have improved packet-layer interpretability.
- Packet-layer metrics should continue to be recorded, but current backend interpretation is still being driven by stage timings.
- The packet boundary is now in a good place for either:
  - a **documentation/interface pass** to strengthen lifecycle/V&V/simulation-credibility planning, or
  - a **project relay** before widening the benchmark matrix further.
