# 2026 03 30 REV3.7 Current Human-Readable Test Report Structure

## Purpose
This document defines the regenerable current human-readable report structure for pyAIGameEngine test reporting.

## Output products
Recommended current report products:
- `reports/current/current_test_report.md`
- `reports/current/current_test_report.html`
- optional later: `reports/current/current_test_report.pdf`

## Report audience
Primary audience:
- modeler / engineering consumer

Secondary audience:
- future AI/RL experimenter
- relay/handoff reader

## Required sections

### 1. Executive summary
Include:
- current development state
- high-level validated capabilities
- main active limitations
- key changes since prior report

### 2. Re-validation summary
Include:
- previous success/failure metrics revalidated
- newly added success/failure metrics
- regressions or newly cleared gates

### 3. Performance summary
Include:
- current benchmark results
- apples-to-apples comparison against prior compatible runs
- notes on whether current values are representative or temporarily distorted by ongoing architecture changes

### 4. Visualization section
Include where available:
- performance-over-time plots
- apples-to-apples comparison plots
- selected rendered simulation snapshots with labels and significance notes
- metric trend graphs

### 5. Projection section
Include:
- provisional complexity-normalized performance estimate
- assumptions used
- whether the current run is representative of projected final performance
- any notable fluctuations and explanation

### 6. Appendix
Include:
- implemented vs unimplemented summary
- test parameters table
- assumptions and justifications
- compatibility notes for trend comparison
- artifact list

## Regeneration rule
This report must always be buildable from archival test data plus referenced stored artifacts.
