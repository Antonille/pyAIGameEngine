# pyAIGameEngine Workflow — REV1.1
Date: 2026-03-26
Revision: 1.1

## Current environment baseline
- Single local `.venv`
- Python 3.12 baseline
- Native optional dependencies are treated as conditional
- PyBullet install is gated behind an MSVC / Windows SDK readiness check

## Immediate workflow rule
1. Run `scripts/windows/install_project.ps1`
2. Run `scripts/windows/run_poc1_validation.ps1`
3. If PyBullet is missing, continue with fallback backend validation instead of blocking POC1
4. Only retry native PyBullet install from a Visual Studio Developer shell

## Native dependency policy
- `pybullet` is optional for the minimally functional POC1 milestone
- `moderngl` remains part of the target stack, but POC1 validation must not fail solely because native physics is unavailable
- Fallback backend exists to preserve architecture progress and reduce local toolchain friction

## Validation expectations
- Baseline imports should pass for: numpy, pyglet, moderngl, gymnasium, pettingzoo, numba, joblib, pydantic
- POC1 benchmark and Gym rollout should pass with `SimpleIntegratorBackend`
- Viewer remains optional/manual
- PyBullet validation is opt-in with `-TryPyBullet`

## Snapshot handling rule
- Sparse snapshot updates should include changed scripts, changed docs, and changed POC files only
- Revision increments when docs/scripts/workflow assumptions change materially
