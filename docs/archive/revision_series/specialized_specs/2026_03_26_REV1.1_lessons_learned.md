# Lessons Learned — REV1.1
Date: 2026-03-26
Revision: 1.1

## Native build lesson
PyBullet may fall back to source build even on Python 3.12. When that happens, success depends on a correctly initialized Visual Studio C++ toolchain and Windows SDK environment.

## Diagnostic lesson
Checking only whether headers exist on disk is not enough. The active shell must expose:
- `cl.exe`
- `INCLUDE`
- `LIB`
- `WindowsSdkDir`
- `UniversalCRTSdkDir`

## Workflow lesson
The install script must never claim success when a package install fails. It must fail fast or explicitly skip optional native packages with a clear warning.

## Architecture lesson
The engine architecture should preserve a fallback backend path so core SoA / plank-tic work can continue even when optional native dependencies are unavailable.
