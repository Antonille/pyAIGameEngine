## ZIP truth-establishment workflow
When a ZIP snapshot is uploaded:
1. use **Python first** to inspect the ZIP contents
2. read `OPEN_THIS_FIRST.md`
3. read `SNAPSHOT_INDEX.json`
4. read current-pointer docs
5. confirm manifest / modification history
6. then inspect the active code files

Do not rely on non-Python ZIP opening paths first when establishing project truth.

## Snapshot/relay workflow guidance
- Full, sparse, and candidate snapshot scripts should generate or refresh:
  - `OPEN_THIS_FIRST.md`
  - `SNAPSHOT_INDEX.json`
  - `docs/CURRENT_ACTIVE_FILE_PATHS.md`
- Relay packaging should confirm that these files exist and point to real files in the package.
- If a candidate snapshot exists, treat it as the default truth source instead of reconstructing state from older sparse artifacts.

## Script validation guidance
Snapshot scripts should be validated on the target shell/runtime, not just assumed portable. Current lessons from this chat family:
- avoid `[System.IO.Path]::GetRelativePath` for Windows PowerShell 5.1
- avoid parameter-default logic that depends on `$PSScriptRoot`
- avoid `TrimStart` char-casting pitfalls for slash cleanup
