# 2026 03 30 REV4.2 Lessons Learned

## Purpose
This document consolidates durable architecture, tooling, benchmark interpretation, testing/reporting, and workflow lessons.

## Already in force
- Separate proof-of-concept folders reduce accidental cross-talk.
- SoA remains the right default experiment for throughput-oriented physics and AI.
- Python 3.12.x single `.venv` remains the practical baseline.
- The engine must preserve a fallback backend path.

## New lessons from accepted testing truth
### Accepted-reporting lesson
- Once archival test data exists and the harness is accepted, the archival record should become the durable testing truth and the human-readable report should be treated as a regenerable derivative.

### Benchmark-interpretation lesson
- The accepted integrated 500 / 1024 evidence no longer justifies the shorthand claim that warmed numba wins unconditionally.
- Current accepted truth is narrower: numpy and numba are currently close on the accepted integrated benchmark family, so performance guidance should avoid stronger claims until more history exists.

### Bottleneck-visibility lesson
- On the accepted 512-body adapter-facing benchmark, AI/scheduling/packet-side orchestration is the largest stage cost.
- That makes AI-facing orchestration a credible near-term optimization target without concluding that physics is solved in the general case.

### Adapter-readiness lesson
- The adapter rollout path is now accepted as exercised evidence, but should still be treated as an early readiness boundary rather than the final canonical RL contract.
