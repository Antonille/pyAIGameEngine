# 2026 03 30 REV4.3 Lessons Learned

## Purpose
Consolidate durable architecture, workflow, testing, and adapter-boundary lessons.

## New lessons from this pass
- The minimal env can be moved toward the packet boundary without committing the project to a public RL API.
- Deterministic fallback-first replay validation is useful as a low-risk boundary-hardening step before broader RL work.
- Input-stage packet application deserves explicit timing surfacing if thin adapters are expected to be trustworthy.
- Replay capture can start as in-memory full-array snapshots; compression and broader replay-file standards can remain deferred.
