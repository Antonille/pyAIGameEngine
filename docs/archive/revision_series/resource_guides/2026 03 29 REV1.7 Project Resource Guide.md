## Why interface work is happening now
The latest local validation shows:
- fallback validation still passes
- warmed Numba can outperform NumPy in the integrated comparison path
- the rigid-body / gravity / field prototype is producing sane outputs

That is enough validated architecture to justify interface-definition documents now, while still being a bit early for a large repo/package split.

## Current active architecture truths
1. Runtime hot-path data should favor **SoA** layouts.
2. POC1 remains **fallback-first** and should not be blocked by PyBullet.
3. AI inference should not be tied to every agent every plank-tic.
4. Repeated object families should use **fixed runtime schemas** and **feature-pack plans** rather than dynamic field fetching.
5. AI scheduling should operate on **model families**, cadence, freshness, and batchability.
6. Reward handling should move toward a **hybrid** model:
   - engine emits canonical objective/event signals
   - AI backends may shape/compose rewards from them
7. Interface control documents should be created before a large code/package refactor.
