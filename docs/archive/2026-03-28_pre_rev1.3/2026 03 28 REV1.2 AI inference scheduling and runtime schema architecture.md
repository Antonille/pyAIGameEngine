# 2026 03 28 REV1.2 AI inference scheduling and runtime schema architecture

This document consolidates the current accepted guidance from the AI inference scheduling prompt and follow-up design work.

## Runtime schema direction
- Use fixed per-family runtime schemas.
- Use CPU-authoritative SoA storage for active POC work.
- Define feature block schemas per model family.
- Use precomputed feature-pack plans rather than ad hoc field fetching.
- Track freshness/version by block, not per scalar.

## AI scheduling direction
- Schedule by model family, not by individual agent call site.
- Each model family defines cadence, max input staleness, max output staleness, stage deadline, and fallback policy.
- Plan CPU->GPU feature movement by packed block.
- Use GPU feature store / output store when the GPU inference path is implemented.
- Overlap CPU work with inference whenever possible.

## POC path
1. Build small CPU-side scheduling structures first.
2. Add feature-pack plans for one repeated object family.
3. Add benchmark comparison tooling and only then begin GPU-oriented scheduling prototypes.
