# Optimized Prompts and Prompt Templates — REV1.0

## Snapshot update prompt template
```text
I am uploading the latest pyAIGameEngine snapshot. Treat it as the candidate new truth.
1. Perform a consistency/sanity check against the previously delivered snapshot and workflow.
2. If it passes, update revision numbers and tracking docs.
3. Make the requested scoped changes.
4. Output either:
   - a sparse snapshot ZIP with only changed/new files, or
   - a full refreshed snapshot ZIP,
   depending on scope.
5. Include near-term workflow summary and the next prompt.
```

## POC implementation prompt template
```text
Continue POC1_SoaFirst.
Scope:
- preserve POC isolation boundaries
- promote nothing to shared root unless explicitly justified
- update POC1 workflow/progress/modification docs
- validate runnable scripts after changes
Deliver:
- code changes
- doc updates
- sparse snapshot ZIP
- benchmark/validation notes
- next prompt
```

## Architecture decision prompt template
```text
For pyAIGameEngine, compare the following architecture options inside isolated POC folders.
For each option:
- state assumptions
- describe likely performance profile
- identify Python integration risks
- recommend what should stay POC-local vs what might become shared later
Update ADR-style notes and provide the next decision prompt.
```

## Best-practice reminder block
Use this block at the end of major prompts:
```text
Before delivering, update revisioned docs, progress notes, modification history, lessons learned, next-step prompts, and any install/snapshot/GitHub scripts touched by this change. If a snapshot is being delivered, ensure it is internally consistent and named with the new revision.
```
