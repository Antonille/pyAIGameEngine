# 2026 03 28 REV1.3 Lessons Learned

1. **Current-truth navigation matters.** Mixed active and historical docs create restart friction. A small current-doc set plus an archive is easier to trust.
2. **Bootstrap scripts must match their docs.** If docs say a script is the first command, the script must be able to create the environment it needs.
3. **Fallback-first architecture paid off.** Preserving a simple integrator backend allowed POC1 to progress despite PyBullet friction.
4. **Full snapshots must be clean.** Caches, `.pyc`, `.egg-info`, and similar byproducts erode confidence in snapshots and complicate diffs.
5. **Install-script ownership must be explicit.** Baseline dependency truth should be documented as owned by the install script until a cleaner packaging/requirements strategy is chosen.
6. **Torch is acceptable as a temporary baseline holdover, but it should stay under review.** Keeping it for now is practical, but it remains a policy decision rather than a settled architecture fact.
