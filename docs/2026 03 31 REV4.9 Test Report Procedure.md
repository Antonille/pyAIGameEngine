# 2026 03 31 REV4.9 Test Report Procedure

## Purpose
Formalize the active Test Report Procedure and explicitly incorporate configurable-interface metadata into testing/report generation when interface validation is part of the active boundary.

## Additional rule now in force
When a configurable-interface manifest is active in the tested runtime path, the testing/reporting system should preserve where applicable:
- manifest ID and version
- manifest hash
- validator version/hash
- startup compatibility-handshake result
- runtime binding mode
- compiled channel summary for the implemented subset

## Current human-readable report expectation
The current report should explicitly state whether:
- the run used the legacy-only path, or
- the run used a startup-validated/compiled configurable-interface subset
and whether that validation passed.

## Durable truth rule remains unchanged
- archival test data remains durable truth
- current human-readable reports remain regenerable from archival data plus referenced artifacts
