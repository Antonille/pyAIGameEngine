# 2026 03 30 REV3.1 Optimized Prompts and Templates

## Durable prompt structure
The most effective prompts in this project usually separate:
1. current-truth anchor
2. pass type
3. constraints / deferrals
4. required outputs
5. exact validation expectations
6. next-step handoff

## Research pass template
Use the current pyAIGameEngine project state as truth.

Pass type:
Research pass.

Task:
Investigate the best package/tool/design options for the specified subsystem before new implementation work.

Requirements:
1. Preserve current validated project truth.
2. Compare realistic candidate options.
3. State what is recommended now, what is plausible later, and what should be deferred.
4. Record tradeoffs, risks, and follow-up actions.
5. Update the relevant docs and provide the next recommended prompt.

## Research sweep template
Use the current pyAIGameEngine project state as truth.

Pass type:
Research sweep.

Task:
Check whether the current architecture path is still well-supported by outside evidence before widening the implementation branch.

Requirements:
1. Review the current-pointer docs first.
2. Formulate search terms around project goals, architecture, implementation rationale, and lessons learned.
3. Search for authoritative sources on similar architectures, goals, runtime strategies, and workflow/tooling plans.
4. Separate findings into recommended now, plausible later, and deferred / not yet.
5. Update working docs with research-aligned lessons, tool candidates, workflow/prompt changes, and the next recommended prompt.

## Architecture/spec pass template
Use the current pyAIGameEngine project state as truth.

Pass type:
Architecture/spec pass.

Task:
Refine or extend the architecture/specification for the requested subsystem.

Requirements:
1. Align with current validated runtime truth and current workflow rules.
2. Do not overcommit to a broader refactor unless justified by validated boundary work.
3. Distinguish simulation-authoritative structures from interface-facing structures where relevant.
4. Identify recommended-now vs deferred items.
5. Update the relevant docs and provide the next recommended prompt.

## Implementation pass template
Use the current pyAIGameEngine project state as truth.

Pass type:
Implementation pass.

Task:
Implement the requested prototype or refinement.

Requirements:
1. Preserve fallback-first architecture and keep PyBullet optional/deferred unless validated work changes that truth.
2. Keep current benchmark/runtime truth intact unless explicitly superseded by new validated results.
3. Name which phase gate or verification objective this implementation pass is primarily serving.
4. Update code, docs, workflow artifacts, and prompts as needed.
5. Deliver exact local validation commands and expected high-level outcomes.
6. Provide the next recommended prompt.

## Gate-clearing implementation prompt pattern
Use the current pyAIGameEngine project state as truth.

Pass type:
Implementation pass.

Task:
Clear the named phase gate without widening unrelated scope.

Requirements:
1. Name the gate being cleared.
2. Implement only what is required to satisfy that gate's acceptance criteria.
3. Deliver exact validation evidence tied to the gate.
4. Keep fallback-first architecture and PyBullet deferral intact unless validated work changes that truth.
5. Update docs, manifest/history, and next-step prompts.
