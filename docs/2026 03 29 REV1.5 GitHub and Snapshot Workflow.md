# 2026 03 29 REV1.5 GitHub and Snapshot Workflow

## Purpose
Keep local repo, snapshot exports, and project-truth docs aligned.

## Standard stopping-point routine
1. run validation commands
2. update current docs and manifest
3. export sparse or full snapshot
4. commit with a meaningful message
5. push only after verifying the local tree is clean

## Snapshot naming
- sparse: `pyAIGameEngine_sparse_snapshot_<version>_<date>.zip`
- full: `pyAIGameEngine_full_snapshot_<version>_<date>.zip`

## Clean snapshot exclusions
Do not include:
- `.venv`
- `__pycache__`
- `.egg-info`
- `.pytest_cache`, `.mypy_cache`, `.ruff_cache`
- numba cache artifacts
- `build`, `dist`, transient output folders
