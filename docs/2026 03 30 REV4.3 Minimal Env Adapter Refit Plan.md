# 2026 03 30 REV4.3 Minimal Env Adapter Refit Plan

## Purpose
Track the near-term refit of the minimal env path toward the packet boundary.

## Decided now
- The minimal env should no longer write directly to `state.action`.
- External control submission should occur by packet through a thin adapter.
- The minimal env should expose:
  - control-path metadata
  - last external packet summary
  - timing snapshot
  - replay snapshot capture / restore hooks

## Temporary scaffolding
- The current packet adapter is local to the POC gym path.
- Actor-family and block-name defaults remain narrow and POC-specific.

## Deferred
- Broad public environment API design.
- Many-world rollout semantics.
- Multi-agent generalized adapter policies.

## Open questions
- Whether the next refit step should move observation staging behind a dedicated feature-block adapter helper.
- Whether external packet submission should later support variable-rate control groups in the env path directly.
