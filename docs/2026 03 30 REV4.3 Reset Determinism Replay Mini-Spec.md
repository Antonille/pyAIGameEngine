# 2026 03 30 REV4.3 Reset Determinism Replay Mini-Spec

## Purpose
Define the current near-term determinism and replay target for fallback-first CPU scenarios.

## Decided now
- Near-term target: deterministic or tightly repeatable fallback-first CPU behavior for:
  - fixed scenario
  - fixed seed
  - fixed action stream
- The minimum replay unit for the current POC is:
  - loop step index
  - full SoA runtime snapshot
- The minimal env adapter now supports deterministic reset by restoring a captured baseline replay snapshot.
- Determinism validation should compare:
  - final state snapshot equality
  - final observation equality
  - total reward equality

## Temporary scaffolding
- Current replay snapshots are Python in-memory dictionaries copied from SoA arrays.
- Replay capture is designed for validation and adapter-facing discipline, not yet for long-term compressed storage.

## Deferred
- Cross-platform bitwise guarantees.
- Stable external replay-file format.
- Full many-world rollout snapshot semantics.

## Open questions
- Whether later replay snapshots should include profiler totals/counts or only simulation state.
- Whether deterministic validation should tolerate floating-point epsilon in future accelerated modes.
