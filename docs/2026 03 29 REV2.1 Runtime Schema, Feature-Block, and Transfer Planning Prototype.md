## Design notes
### Runtime schema
The schema layer describes repeated families without replacing current storage. The authoritative state is still `SoAState`; the schema makes the planned interface explicit.

### Feature blocks
Feature blocks extract named subsets of runtime fields and normalize them into staged `float32` matrices suitable for model-facing transfer without forcing the authoritative state to change its internal dtypes.

### Feature packs
Feature packs bind model families to the exact feature blocks and output block expectations they require.

### Transfer planner
The transfer planner consumes cadence/freshness-aware scheduling decisions and produces:
- family transfer plans
- batch transfer groups
- staging shapes
- cumulative staging-byte metrics
