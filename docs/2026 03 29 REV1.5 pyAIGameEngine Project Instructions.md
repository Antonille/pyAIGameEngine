## Core project concepts and vocabulary
Use the following terms consistently unless later revised:
- **plank-tic**: smallest unit of simulation/game time. All in-simulation time differences are integer numbers of plank-tics.
- **plank-pix**: smallest distance unit.
- **voxellino**: smallest scalar-valued volume element.
- **voxel**: smallest cubic boundary/surface element used to describe geometry.
- **coordinate system**: a packed set of voxels.
- **master stationary reference frame**: the coordinate system containing the extent of all others.
- **shapes/objects**: entities defined relative to coordinate systems, with surface/boundary and property descriptions.

## Project direction
The engine architecture should prioritize:
- scalable architecture over short-lived hacks
- efficient plank-tic update flow
- efficient comparison/rendering of object descriptions into a common coordinate space for interaction calculation
- efficient repacking of results back into object-relative representations
- vectorization and later kernel migration where beneficial
- preserved compute headroom for AI behavior
- proof-of-concept isolation before promotion into shared engine layers

The project may begin with simple mechanics, but every implementation should be evaluated for how well it supports later complexity without forcing a major refactor.


## Frame and physics direction
- Simulation-bearing scene/reference frames remain rigid and inertial in the current POC assumptions.
- Body-local / shape-local frames are rigidly attached to bodies and may be non-inertial; they are used for geometry, material/property layout, local inertia, and later interaction samples rather than as primary force-balance solve frames.
- Master gravity is time-varying and owned by the master stationary reference frame; child simulation frames inherit and update their local gravity vectors via rigid transform propagation.

## AI direction
A major long-term goal is modular AI integration. The engine should leave enough compute headroom for AI-driven behaviors such as adversarial movement, tactical inference from partial observations, fog-of-war reconstruction, and learned counter-play.

Current accepted AI architecture direction:
- AI inference should not be tied to every agent every plank-tic.
- Repeated object families should use fixed runtime schemas and feature-pack plans rather than dynamic field fetching.
- Scheduling should operate on model families, cadence, freshness, and batchability.
- CPU-authoritative SoA storage remains the current truth for active POC work.
