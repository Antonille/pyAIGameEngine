# 2026 03 30 REV4.5 Lessons Learned

## Purpose
Capture durable project and workflow lessons after the cleanup/consolidation pass.

## Best practices strengthened
- Keep the active repo/documentation surface intentionally small.
- Archive superseded revision series instead of letting them compete with current truth.
- Remove transient/editor/runtime byproducts before publishing snapshots or repo truth.
- Preserve the lesson-action register as the thin implementation layer on top of narrative lessons.
- Treat full snapshots as authoritative only after reconciliation against later sparse work.

## New lesson from this pass
### Best practice now adopted
A consolidation pass should produce both:
- a cleaned working tree ready for GitHub truth
- an archive structure that preserves history without leaving the active surface crowded and ambiguous

### Best practice now adopted
If a file exists only to explain how to apply a sparse refresh, it should not survive into durable repo truth unless it has ongoing operational value.

### Best practice now adopted
Concatenation is useful when a historical series becomes noise at the active surface, especially for repeated modification-history and state-manifest documents.
