# 2026 03 30 REV4.5 GitHub and Snapshot Workflow

## Purpose
Clarify how this cleaned full snapshot should be promoted into GitHub truth.

## Current rule
- GitHub main is the desired long-term repo-truth surface.
- This cleaned full snapshot may temporarily be ahead of GitHub and should then be treated as bounded `snapshot-truth` pending review and commit.

## Promotion workflow for this snapshot
1. Replace the local repo contents with the reviewed full snapshot.
2. Run validation commands.
3. Review `git status` / `git diff --stat`.
4. Commit with a cleanup/consolidation message.
5. Push to GitHub main.
6. Recheck critical public-main entry points.

## Important note
This doc does **not** claim the legacy latest-snapshot selector is fixed.
That remains a separate open-risk implementation pass.
