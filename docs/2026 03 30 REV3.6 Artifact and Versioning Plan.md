# 2026 03 30 REV3.6 Artifact and Versioning Plan

## Purpose
This document defines the lightweight artifact/versioning discipline needed before pyAIGameEngine widens into broader experiment, simulation, and AI/RL work.

## Artifact classes currently in scope
- candidate snapshots
- benchmark configurations
- scenario definitions / bundles
- model artifacts later
- hyperparameter / experiment configs later
- adapter-facing run metadata now

## Gate A artifact fields required
Adapter-facing runs should record at minimum:
- `scenario_version`
- `adapter_version`
- `seed`
- `baseline_policy_id`
- `mode`

Recommended near-term additions:
- `scenario_bundle_id`
- `observation_contract_version`
- `action_contract_version`

## File naming guidance for Gate A era
Recommended lightweight conventions:
- scenario bundle: `scenario_bundle_<name>_v<major.minor>.json`
- adapter config: `adapter_config_<name>_v<major.minor>.json`
- baseline policy ID: `baseline_<name>_v<major.minor>`

## Current recommendation
Do not build a heavy artifact-management system yet. Do start treating adapter-facing mode tags, seeds, scenario versions, and baseline policy IDs as first-class recorded artifacts.
